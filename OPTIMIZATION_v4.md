# Car Scout 优化策略 v4.0
# 从"数据收集"转向"决策支持"

## 🎯 核心理念转变

**旧模式**: 收集所有车 → 慢慢筛选  
**新模式**: 只抓好车 → 快速决策 → 立即行动

---

## 1️⃣ 数据清理策略

### 保留规则（只存这些）
| 条件 | 说明 |
|------|------|
| Flip Score ≥ 60 | A级和S级车 |
| 跟进列表 | 用户标记的车 |
| 近3天上架 | 新车观察期 |
| 已售检测 | 跟进池车辆状态 |

### 清理频率
- **每日**: 自动删除 Flip Score < 50 且超过7天的车
- **每周**: 深度清理，只保留高分车 + 跟进池

---

## 2️⃣ 高频扫描策略

### 分层扫描

| 层级 | 车型 | 频率 | 说明 |
|------|------|------|------|
| **T0** | Corolla, Vitz, Honda Fit | 每30分钟 | 周转最快，重点监控 |
| **T1** | RAV4, Swift, Demio | 每小时 | 利润较高 |
| **T2** | Wish, Prius, Aqua | 每2小时 | 补充扫描 |

### 时段加权
- **早高峰** (7-9点): 2倍频率（上班路上刷手机的人多）
- **午休** (12-13点): 正常频率
- **晚高峰** (17-21点): 2倍频率（下班后发布高峰）
- **深夜** (22-7点): 停止扫描

### 快速筛选清单

看到新车时，5秒内判断：

1. **价格** ≤ $5,000? 
2. **年份** ≥ 2005?
3. **Flip Score** ≥ 65?
4. **里程** ≤ 160k 或未知?
5. **卖家** 是个人?

全部符合 = **立即联系**

## 4️⃣ 已售检测优化

### 跟进池管理

| 车辆 | 状态 | 行动 |
|------|------|------|
| Flip Score ≥ 75 | 高优先级 | 每天检查是否已售 |
| 50 ≤ Flip < 75 | 中优先级 | 每3天检查 |
| 已联系卖家 | 标记 | 手动更新状态 |
| 超过7天无更新 | 自动移除 | 清理数据 |
