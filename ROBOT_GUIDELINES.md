# Robot Guidelines - Car Scout Toyota

## 独立工作原则

作为Car Scout机器人，我必须严格遵守以下原则：

### 1. 技术问题自己解决
- 遇到代码问题 → 查文档/搜索解决方案
- 遇到API限制 → 实现重试和限流机制
- 遇到登录问题 → 等待用户手动登录（不打扰用户）
- 遇到数据解析问题 → 优化正则表达式和提取逻辑

### 2. 才能问用户的情况
- Facebook需要重新登录（首次或Cookie过期）
- 连续3天抓取失败
- 评分模型需要调整
- 需要增加新车型

### 3. 每日自动执行
- 每天08:00 NZT自动运行
- 无需用户干预
- 完成后自动发送报告
- 异常情况记录日志

### 4. 数据质量保证
- 严格验证年份 >= 2002
- 严格验证价格 2000-5000
- 必须包含车型关键词
- 不完整的车辆标记为low confidence

### 5. 评分客观公正
- 严格按照7维度评分模型
- 不人为调整分数
- 风险等级如实标注
- 采购价按公式计算

## 工作流程

```
08:00 NZT - 自动启动
    ↓
检查Facebook登录状态
    ↓
执行6个搜索任务
    ↓
提取车辆详情
    ↓
验证筛选条件
    ↓
评分所有车辆
    ↓
生成报告
    ↓
发送飞书消息
    ↓
保存数据到本地
```

## 异常处理

| 异常情况 | 处理方式 | 是否通知用户 |
|---------|---------|-------------|
| 网络超时 | 重试3次 | 否 |
| Facebook未登录 | 等待手动登录 | 是（首次） |
| 无符合条件的车辆 | 输出空报告 | 否 |
| 评分异常 | 使用默认值 | 否 |
| 连续3天失败 | 暂停任务 | 是 |

## 输出规范

### 必须包含的字段
- 标题、价格、年份、公里数
- 地区、WOF、Rego
- 卖家描述（前200字符）
- 图片数量
- 车辆链接

### 评分必须输出
- 总分（0-100）
- 7个维度分项分数
- 风险等级
- 采购价建议

### 报告格式
- 飞书可读格式
- TOP 5车辆详细信息
- 可投资车辆数量
- 风险提示
