# 🔧 Render 部署失败排查指南

## 问题现象
- 网站显示黑色页面
- Render 显示 "Deploy Failed"

## 排查步骤

### 步骤1：查看Render日志

1. 打开 https://dashboard.render.com
2. 点击 **car-scout** 服务
3. 点击 **Logs** 标签
4. 查看最新的错误信息

### 步骤2：常见错误及修复

#### 错误A：缺少依赖
**日志显示：** `Cannot find module 'xxx'`

**修复：**
1. 点击 **Settings**
2. 找到 **Build Command**
3. 确保是：`npm install`
4. 点击 **Manual Deploy** → **Clear build cache & deploy**

#### 错误B：启动命令错误
**日志显示：** `Error: Cannot find module` 或路径错误

**修复：**
1. 点击 **Settings**
2. 找到 **Start Command**
3. 修改为：`node src/start.js`
4. 保存并重新部署

#### 错误C：端口未监听
**日志显示：** `No open ports detected`

**修复：**
检查 `src/web-server.js` 是否正确监听 `process.env.PORT`

#### 错误D：文件缺失
**日志显示：** `Cannot find module '/opt/render/project/src/xxx'`

**修复：**
确保所有文件已上传到GitHub，然后重新部署

### 步骤3：强制重新部署

1. 在Render Dashboard中
2. 点击 **Manual Deploy**
3. 选择 **Clear build cache & deploy**
4. 等待重新构建

### 步骤4：如果还是失败

**方案A：删除重建**
1. 删除 car-scout 服务
2. 重新创建（选择GitHub仓库）
3. 配置：
   - Build: `npm install`
   - Start: `node src/start.js`

**方案B：简化部署**
只部署静态HTML页面：
1. 在Render创建 **Static Site**
2. 上传 `车辆管理看板.html`
3. 这样至少能看到界面
