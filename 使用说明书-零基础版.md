# 🚗 Car Scout - 使用说明书

## 不需要懂代码！只需双击文件即可！

---

## 📂 文件说明

文件夹里有5个批处理文件（.bat），双击即可运行：

| 文件 | 作用 | 使用频率 |
|------|------|---------|
| **1-开始抓取.bat** | 抓取Facebook车辆数据 | 每天1次 |
| **2-生成报告.bat** | 生成今日报告 | 每天1次 |
| **3-查看待办.bat** | 查看今天要做的事情 | 随时查看 |
| **4-打开看板.bat** | 打开可视化看板 | 随时查看 |
| **5-生成看车清单.bat** | 生成看车检查清单 | 需要看车时 |

---

## 🚀 每天使用步骤（只需3步）

### 第1步：抓取车辆（每天早上）
1. 双击 **1-开始抓取.bat**
2. 等待浏览器打开
3. 如果要求登录Facebook，输入账号密码
4. 等待抓取完成（约10分钟）
5. 按任意键关闭窗口

### 第2步：生成报告
1. 双击 **2-生成报告.bat**
2. 等待报告生成
3. 按任意键自动打开报告文件夹

### 第3步：查看报告
1. 双击 **4-打开看板.bat**
2. 双击最新的HTML文件（例如：dashboard_2024-02-27.html）
3. 在浏览器中查看所有车辆

## 📝 看车时使用

### 预约看车前
1. 双击 **5-生成看车清单.bat**
2. 输入车辆ID（从看板中复制）
3. 自动生成看车清单
4. 打印或用手机查看清单

### 看车时
- 按照清单逐项检查
- 记录车况和问题
- 拍照留档

## 🔄 如何更新车辆状态？

目前需要通过命令行更新（后续会制作GUI界面）：

1. 打开命令提示符（CMD）
2. 输入：`npm run scout advance [车辆ID] [新状态]`

例如：
```
npm run scout advance fb_123 contacted
```

状态选项：
- `contacted` - 已联系
- `viewing` - 预约看车
- `inspecting` - 车检中
- `purchased` - 已购买
- `cleaning` - 清理中
- `reselling` - 已上架
- `sold` - 已售出

## ⚠️ 常见问题

### Q: 双击没反应？
A: 确保已安装Node.js。如果未安装，先运行 `安装依赖.bat`

### Q: 为什么每次都要登录Facebook？
A: 首次登录后会自动保存，如果提示过期需要重新登录

### Q: 数据会丢失吗？
A: 不会，所有数据自动保存在 `database/vehicles.json`

### Q: 如何备份数据？
A: 复制 `database/vehicles.json` 到其他位置即可
